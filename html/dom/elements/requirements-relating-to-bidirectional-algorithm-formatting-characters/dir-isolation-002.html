  • In a LTR context, the Unicode Bidirectional Algorithm causes a sequence of Hebrew letters in memory, separated only by weak-directional characters, to display from RTL. This test puts a span around each letter, with the dir attribute set, in order to isolate the letters from each other. This means that they should be ordered LTR, per the surrounding context.
  • It repeats this test for a RTL and LTR surrounding context.
  • Reference text is drawn in red behind the test text, and will show through if the order is incorrect. (&#x202d; is LRO, &#x202d; is PDF, א is Hebrew alef.)
